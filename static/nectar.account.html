class nectar.account.Account(account, full=True, lazy=False, blockchain_instance=None, **kwargs)ÔÉÅ

Bases: BlockchainObject

This class allows to easily access Account data

Parameters:

  • account (str) ‚Äì Name of the account

  • blockchain_instance (Steem/Hive) ‚Äì Hive or Steem instance

  • lazy (bool) ‚Äì Use lazy loading

  • full (bool) ‚Äì Obtain all account data including orders, positions, etc.

  • hive_instance (Hive) ‚Äì Hive instance

  • steem_instance (Steem) ‚Äì Steem instance

Returns:

Account data

Return type:

dictionary

Raises:

nectar.exceptions.AccountDoesNotExistsException ‚Äì if account does not exist

Instances of this class are dictionaries that come with additional methods (see below) that allow dealing with an account and its corresponding functions.

>>> from nectar.account import Account
>>> from nectar import Hive
>>> from nectar.nodelist import NodeList
>>> nodelist = NodeList()
>>> nodelist.update_nodes()
>>> stm = Hive(node=nodelist.get_hive_nodes())
>>> account = Account("gtg", blockchain_instance=stm)
>>> print(account)
<Account gtg>
>>> print(account.balances)

Note

This class comes with its own caching function to reduce the load on the API server. Instances of this class can be refreshed with Account.refresh(). The cache can be cleared with Account.clear_cache()

blog_history(limit=None, start=-1, reblogs=True, account=None)ÔÉÅ

Stream the blog entries done by an account in reverse time order.

Note

RPC nodes keep a limited history of entries for the user blog. Older blog posts of an account may not be available via this call due to these node limitations.

Parameters:

  • limit (int) ‚Äì (optional) stream the latest limit blog entries. If unset (default), all available blog entries are streamed.

  • start (int) ‚Äì (optional) start streaming the blog entries from this index. start=-1 (default) starts with the latest available entry.

  • reblogs (bool) ‚Äì (optional) if set True (default) reblogs / resteems are included. If set False, reblogs/resteems are omitted.

  • account (str) ‚Äì (optional) the account to stream blog entries for (defaults to default_account)

blog_history_reverse example:

from nectar.account import Account
from nectar import Steem
from nectar.nodelist import NodeList
nodelist = NodeList()
nodelist.update_nodes()
stm = Steem(node=nodelist.get_hive_nodes())
acc = Account("steemitblog", blockchain_instance=stm)
for post in acc.blog_history(limit=10):
    print(post)

get_account_history(index, limit, order=-1, start=None, stop=None, use_block_num=True, only_ops=[], exclude_ops=[], raw_output=False)ÔÉÅ

Returns a generator for individual account transactions. This call can be used in a for loop.

Parameters:

  • index (int) ‚Äì first number of transactions to return

  • limit (int) ‚Äì limit number of transactions to return

  • start (int, datetime) ‚Äì start number/date of transactions to return (optional)

  • stop (int, datetime) ‚Äì stop number/date of transactions to return (optional)

  • use_block_num (bool) ‚Äì if true, start and stop are block numbers, otherwise virtual OP count numbers.

  • only_ops (array) ‚Äì Limit generator by these operations (optional)

  • exclude_ops (array) ‚Äì Exclude thse operations from generator (optional)

  • batch_size (int) ‚Äì internal api call batch size (optional)

  • order (int) ‚Äì 1 for chronological, -1 for reverse order

  • raw_output (bool) ‚Äì if False, the output is a dict, which includes all values. Otherwise, the output is list.

Note

only_ops and exclude_ops takes an array of strings: The full list of operation ID‚Äôs can be found in nectarbase.operationids.ops. Example: [‚Äòtransfer‚Äô, ‚Äòvote‚Äô]
